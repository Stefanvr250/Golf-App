# GolfApp

A Progressive Web App (PWA) golf companion for South African golfers: course GPS,
detailed scoring, handicap tracking, tournaments, leagues, predictions, and social play.

Built with **Next.js 14 (App Router)**, **Supabase** (PostgreSQL + PostGIS, Auth,
Realtime, Storage), **MapLibre GL JS**, **TailwindCSS**, and **shadcn/ui**.

See full specs in [`Documentation/golf-app/`](Documentation/golf-app/).

## Prerequisites

- Node.js 18+ (tested on Node 24)
- A free [Supabase](https://supabase.com) project
- A free [MapTiler](https://www.maptiler.com) API key (satellite tiles)

## Setup

1. **Install dependencies**

   ```bash
   npm install
   ```

2. **Create a Supabase project** (Task 1.3)
   - Create a new project at https://supabase.com.
   - In the SQL editor (or via the Supabase CLI), run the migrations in order:
     - `supabase/migrations/0001_initial_schema.sql` (enables PostGIS + pg_trgm, creates all tables)
     - `supabase/migrations/0002_rls_policies.sql` (Row Level Security policies)

3. **Configure environment variables**
   - Copy `.env.local.example` to `.env.local` and fill in the values:

   ```bash
   cp .env.local.example .env.local
   ```

   | Variable | Where to find it |
   |----------|------------------|
   | `NEXT_PUBLIC_SUPABASE_URL` | Supabase → Project Settings → API |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase → Project Settings → API |
   | `SUPABASE_SERVICE_ROLE_KEY` | Supabase → Project Settings → API (server only) |
   | `NEXT_PUBLIC_MAPTILER_KEY` | MapTiler account dashboard |
   | `NEXT_PUBLIC_APP_URL` | App base URL (e.g. `http://localhost:3000`) |

4. **Run the dev server**

   ```bash
   npm run dev
   ```

   Open http://localhost:3000.

## Scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start the dev server (PWA disabled in dev) |
| `npm run build` | Production build (generates the service worker) |
| `npm run start` | Serve the production build |
| `npm run lint` | Run ESLint |

## Notes

- The PWA service worker is generated by `next-pwa` only in production builds
  (`public/sw.js`, git-ignored).
- App icons currently use a single SVG (`public/icons/icon.svg`). For best
  iOS/Android home-screen support, add rasterized PNG icons (192px, 512px,
  maskable) and re-add them to `public/manifest.json`.
- On Windows PowerShell, if `npm` is blocked by the execution policy, use
  `npm.cmd` (e.g. `npm.cmd run dev`).

## Project structure

```
src/
├── app/            # App Router pages, layouts, API routes
├── components/
│   ├── ui/         # shadcn/ui base components
│   └── layout/     # App shell (nav, online status)
├── hooks/          # React hooks (e.g. use-toast)
├── lib/
│   ├── supabase/   # Browser/server clients + auth middleware
│   └── validations/# Zod schemas
└── middleware.ts   # Supabase auth session refresh
supabase/migrations # SQL schema + RLS policies
```
