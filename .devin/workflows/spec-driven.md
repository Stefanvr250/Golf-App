description: Orchestrator for Spec-Driven Development (SDD). Use to understand the full workflow or get guidance on which step to run next.
---

Use the @spec-driven skill to see the full SDD workflow and determine which step command to use. SDD is split into four step commands so you can review and iterate at each checkpoint.

Step commands:
- `/specify <feature idea>` — Grill the idea + create the PRD
- `/spec-plan <feature-name>` — Create the Technical Spec from the PRD
- `/spec-tasks <feature-name>` — Create the Implementation Plan and Task List from the Technical Spec
- `/spec-build <feature-name>` — Read the full spec chain and implement the tasks
